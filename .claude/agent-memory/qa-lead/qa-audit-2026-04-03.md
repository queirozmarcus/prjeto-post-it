| Unit | 69 | 86% | 70% |

**Diagnóstico:** ✅ Proporção saudável

- Unit tests dominam (86%) — correto
- Integration tests adequados (14%) — poderia chegar a 20%, mas 14% é aceitável
- E2E tests inexistentes — **não necessários no momento** (não há múltiplos serviços, não há UI automatizada)

**Nota sobre E2E/Contract:**
- Contract tests (Pact, Spring Cloud Contract) são para integrações entre serviços. Este projeto é monólito — não aplicável.
- E2E tests (Playwright, Selenium) são para UI. Frontend é Vue SPA separado — não no escopo do backend.
- AuthIntegrationTest funciona como pseudo-E2E (full HTTP flow com banco real).

**Conclusão:** Pirâmide está correta para arquitetura atual.

   - Risco: Baixo (unit tests cobrem lógica, integration testa infra)
   - Impacto: Baixo (risco de regressão apenas em queries específicas)
